## Bug Description

Installation fails with:
```
sqlalchemy.exc.ProgrammingError: relation "ix_patch_repos_name" already exists
```

## Root Cause

In `backend/models/db_models.py`, the `PatchRepo.name` column has both `unique=True` and `index=True`. SQLAlchemy creates an implicit index for `unique=True`, and the explicit `index=True` creates a duplicate.

## Fix Applied

Removed `index=True` since `unique=True` already creates an implicit index.

## Impact

- Customers installing v2.0.17 and earlier experience database initialization failure
- System left in broken state after partial installation

## Customer Workaround

For existing installations, drop the duplicate index:
```sql
DROP INDEX IF EXISTS ix_patch_repos_name;
```
