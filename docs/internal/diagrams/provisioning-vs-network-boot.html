Provisioning vs Network Boot

Internal reference explaining the exact difference between the production-ready Provisioning Center (enrolled-host imaging) and the managed-relay Network Boot workspace (bare-metal pre-agent rollout). Shows how both capabilities fit into a single enterprise operating model.

1 Current Provisioning Center Flow

Live production path for already-enrolled endpoints — golden image capture, store, and restore.

flowchart TD A["Operator opens Provisioning Center"] --> B["Choose registered source host"] B --> C["Capture full-system image"] C --> D["Store golden image on PatchMaster server"] D --> E["Create provisioning template"] E --> F["Choose registered target hosts"] F --> G["Queue rollout through operations queue"] G --> H["Target agents download and restore image"] H --> I["PatchMaster records per-target results"]

2 Managed Network Boot Flow

Bare-metal workspace for managed-relay deployment — no agent required on the target yet.

flowchart TD A["Operator opens Network Boot"] --> B["Create boot network"] B --> C["Define addressing, relay binding, BIOS and UEFI boot files"] C --> D["Create boot profile"] D --> E["Choose install mode and install source"] E --> F["Optional: link stored golden image template"] F --> G["PatchMaster generates iPXE script"] G --> H["PatchMaster generates answer file"] H --> I["Operator validates rollout design"] I --> J["Relay bundle prepared for deployment"] J --> K["Bare-metal endpoint boots and enrolls"]

3 Managed Relay Workflow

Complete operator flow from network design through first-boot enrollment.

flowchart LR A["Define networks\nand relay binding"] --> B["Publish boot artifacts\nand answer files"] B --> C["Install and validate\nmanaged relays"] C --> D["Run bare-metal rollout\nand first-boot enrollment"]

4 Side-By-Side Capability Comparison

Why current provisioning and PXE-style rollout are related but not identical.

✅ Current Provisioning Center

  • Requires enrolled agent on target
  • Uses snapshot + restore APIs
  • Best for reimage and recovery
  • Works on live, online fleet hosts
  • Golden image stored on PatchMaster server

🌐 Managed Network Boot

  • Designed for pre-agent bare-metal hosts
  • Uses iPXE + answer file generation
  • Includes relay bundle + unattended boot
  • First-boot enrolls agent automatically
  • Supports BIOS and UEFI targets

6 Operator Instructions

Use Provisioning Center When
  1. Target machine is already enrolled with PatchMaster agent.
  2. You need live golden-image reimage, recovery, or branch refresh.
  3. You want rollout execution today, not future transport design.
Use Network Boot When
  1. You want to define boot networks and profile standards.
  2. You want to preview iPXE and answer-file artifacts.
  3. You want to prepare managed relay services and run pre-agent rollout from the same workspace.
Safe Product Message
  1. PatchMaster already provides production-ready stored-image provisioning for enrolled hosts.
  2. PatchMaster now also provides a managed-relay network-boot workspace.
  3. Relay preparation, boot artifact delivery, and first-boot enrollment are part of the live operator flow.
