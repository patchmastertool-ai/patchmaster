License & Release Architecture

Internal reference for the release flow, offline PM2 license model, installation paths, trust boundaries, and exact operator deployment instructions. Updated to reflect Ed25519 signing, X25519 encryption, and the PostgreSQL 17 / Argon2 vendor portal migration.

5 Runtime License Flow

End-to-end flow from Vendor customer record creation through final PM2 license activation in PatchMaster.

flowchart TD A["Vendor records customer and purchase"] --> B["Issue POC or testing license"] B --> C["Customer activates testing license in PatchMaster"] C --> D["Customer shares verified hardware or MAC ID with Vendor"] D --> E["Vendor updates customer record with final binding ID"] E --> F["Vendor generates final bound PM2 license"] F --> G["Customer activates final PM2 in PatchMaster"] H["PatchMaster runtime license bundle"] --> I["Verify Ed25519 signature\nand decrypt claims locally"] G --> I I -->|Valid| J["License accepted"] I -->|Invalid| K["Signature, decrypt, or binding validation failed"]
PatchMaster verifies and decrypts licenses locally using its bundled runtime material. The Vendor portal does not need to be reachable at runtime. Hardware or MAC ID binding is recorded in the Vendor portal before the final license is generated.

9 Operator Instructions

Internal Deployment
  1. Run bash scripts/build-stable.sh.
  2. Keep dist/private/patchmaster-license-authority.env private.
  3. Send dist/patchmaster-product-2.0.0.tar.gz to the PatchMaster server.
  4. Send vendor/dist/patchmaster-vendor-2.0.0.tar.gz to the Vendor server.
PatchMaster Server
  1. Extract the customer product tarball.
  2. Run bash packaging/install-bare.sh --with-monitoring.
  3. PatchMaster installs with the bundled runtime license bundle.
Vendor Server
  1. Extract the vendor tarball aligned to your license authority set.
  2. Run bash install-vendor.sh.
  3. Vendor installs with the private signing authority.
Release Hygiene Rules
  • Never give patchmaster-license-authority.env to customers.
  • Keep product and vendor packages on the same license authority set.
  • Use a testing license first for POC hardware binding.
  • Record hardware/MAC ID in Vendor portal before the final license.
  • Vendor internet exposure is NOT required for license verification.
